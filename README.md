В даному проекті було досліджено, що станеться, якщо змінну буде випадково використано як показчик, коли вона їм не являється.

Напишемо просту програму з однією змінною типу `int` та один показчик типу `int *`, `num` та `ptr` відповідно.

![](Photos/1.png)

Далі ми призначаємо показчику `ptr` значення `num`, але замість передавання адреси змінної `&num`, ми передаємо саме значення, тобто `ptr = num`.

![](Photos/2.png)

Таким чином `ptr` тепер вказує на пам'ять `3`, що не є дійсною\існуючою, через що і викликається аварійне завершення `Segmentation Fault`.
